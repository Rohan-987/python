# Encapsulation, Abstraction, Inheritance , Polymorphism

# Instance Variable - unique to every object. for ex: every car has different color
#                     but same number of wheels
# Class variable - shared by all objects. 
 
# Encapsulation is the practice of bundling data (variables) and the methods 
# that work on that data into a single unit (a class). It also involves hiding 
# the internal details so they can't be changed accidentally from the outside.
# Example: A BankAccount class has a balance. You shouldn't be able to change 
# the balance directly (e.g., account.balance = 1000000). Instead, you use methods 
# like deposit() or withdraw() to change it safely.

# Abstraction means hiding the complex background logic and only showing the 
# essential features to the user. You care about what an object does, not how 
# it does it.
# Example: When you use a TV remote, you press the "Volume Up" button. 
# You don't need to know how the circuit board processes the signal; you
# just need the button (the interface).

# Inheritance allows a new class (the Child) to adopt the attributes and methods
# of an existing class (the Parent). It helps you avoid writing the same code twice.
# Example: A Car class and a Truck class can both inherit from a general Vehicle 
# class because they both have wheels and engines.

# Polymorphism allows different classes to be treated as if they are the same "type,"
# usually because they share the same method names but perform them differently.

# Example: Both a Dog and a Cat have a speak() method. But the Dog says "Woof" and 
# the Cat says "Meow." You can call .speak() on any animal without needing to know 
# exactly what it is first.
